삼성 기출 1번 유형만 풀다가, DP와 최적화가 함께 나오는 2번 문제도 풀어보았다.

여러 작업 중 4번 블렌딩 작업은 목표 향도 K를 정확히 만들기 위한 최소 향료 개수를 구하는 문제였고, 재귀 함수와 메모이제이션을 이용한 DP 방식으로 비교적 자연스럽게 해결할 수 있었다.

반면 5번 향수 구성 작업은 탑노트, 미들노트, 베이스노트 총 3개의 향료를 고르는 문제였는데, 처음에는 중복 순열 방식으로 완전탐색을 적용했다. 하지만 가능한 향료 종류가 많아지면 3중 반복문 구조가 되어 최대 약 1000 × 1000 × 1000 수준의 연산이 발생했고, 이로 인해 시간 초과가 발생했다.

이를 해결하기 위해 세 번째 향료까지 직접 탐색하지 않고, 특정 향도 이상을 가진 살아있는 향료의 개수를 역누적합 테이블에 미리 저장했다. 이후 첫 번째와 두 번째 향료만 2중 반복문으로 고르고, 필요한 세 번째 향료의 최소 향도를 계산한 뒤 역누적합 값을 더하는 방식으로 최적화했다.

이번 문제를 통해 단순히 DP를 적용하는 것뿐만 아니라, 경우의 수 문제에서는 완전탐색의 차원을 줄이는 사고가 중요하다는 것을 느꼈다. 특히 3개를 모두 직접 고르지 않고, 일부는 미리 계산한 테이블로 대체하면 시간복잡도를 크게 줄일 수 있다는 점이 핵심이었다.
