## 문제 풀이

처음에는 스택으로 `(`, `)`, `?`를 하나씩 처리하려고 했다.

근데 `?`를 만났을 때 이걸 `(`로 쓸지 `)`로 쓸지 바로 정하면, 뒤에 뭐가 나올지 모르기 때문에 꼬였다.
그래서 `?`를 확정하지 않고, 현재 가능한 열린 괄호 개수의 최소와 최대를 저장하는 식으로 풀었다.

* `min_open` : 지금까지 `?`를 최대한 `)`로 썼을 때 남아 있는 `(`의 개수
* `max_open` : 지금까지 `?`를 최대한 `(`로 썼을 때 남아 있는 `(`의 개수

문자마다 값은 이렇게 바뀐다.

```cpp
'(' : min_open++, max_open++
')' : min_open--, max_open--
'?' : min_open--, max_open++
```

`?`는 `)`가 될 수도 있으니까 최소값은 1 감소하고,
`(`가 될 수도 있으니까 최대값은 1 증가한다.

중간에 `min_open`이 음수가 되면 0으로 바꿔준다.

```cpp
min_open = max(min_open, 0);
```

열린 괄호 개수가 음수일 수는 없고,
`?`를 다른 방식으로 사용해서 0인 경우가 가능할 수 있기 때문이다.

그리고 `max_open < 0`이 되면 바로 `No`다.

`max_open`은 `?`를 전부 `(`로 사용한 가장 유리한 경우인데,
이 값까지 음수가 됐다는 것은 앞에 나온 `)`를 막을 방법이 없다는 뜻이다.

예를 들면

```text
?))
```

에서 `?`를 `(`로 바꿔도

```text
())
```

가 되기 때문에 마지막 `)`는 처리할 수 없다.

마지막에는 `min_open == 0`인지 확인한다.

`min_open`은 `?`를 최대한 `)`로 사용한 경우인데도 남아 있는 `(`의 개수다.
이 값이 0보다 크면 기록이 끝났는데도 닫히지 않은 `(`가 남아 있다는 뜻이다.

또 전체 길이가 홀수면 `(`와 `)`의 개수를 똑같이 만들 수 없기 때문에 처음부터 `No` 처리했다.

```cpp
if (n % 2 == 1) {
    cout << "No";
    return 0;
}
```

결국 핵심은 `?`를 바로 결정하지 않고,

```text
현재 가능한 열린 괄호 개수의 범위
[min_open, max_open]
```

를 계속 들고 가는 것이다.

`max_open`으로 앞에서 해결할 수 없는 `)`가 생겼는지 확인하고,
마지막에 `min_open`이 0인지 확인해서 끝까지 닫히지 않은 `(`가 있는지 확인했다.
