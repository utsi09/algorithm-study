또한 이 문제 핵심은 

1. 정사각형 찾기

2. 회전하기

인데, 정사각형 찾기는 각 npc 마다 탈출구까지의 넓이나 높이를 기준으로 윈도우 크기를 설정하고 0,0 부터 

불린으로 npc가 들어있는지 탈출구가 들어있는지, 맵을 벗어나지않는지 윈도우 슬라이딩으로 체크해서 찾았다

size가 더 작아지면 r,c 위치 와 상관없이 다시 갱신했는데 해설 코드를 보니 더 간단한 방식이 있었다

단순히 사이즈를 2~n 으로 0,0~n,n 으로 하는 방식이다 size가 작을때부터, 0,0부터 하니까 더 작은 경우가 나왔을때의 처리를 할 필요가없다

또한 회전하기는 큰 맵에서 국소 부분만 회전할때에 사용 하는 방식인데,

현재 좌상단의 좌표를 빼서 로컬 x,y로 변환하고 그 상태에서 회전 (로컬좌표계가 아닐 경우, 회전할때 사용하는 맵의 사이즈와 호환되지않아 원하는 방식처럼 회전되지 않는다) 한다.

그렇게 회전된 로컬 회전 맵을 다시 좌상단 좌표를 더해 글로벌 회전맵으로 만든다
